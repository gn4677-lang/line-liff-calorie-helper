# 台北外食場景與地點上下文

## Goal

支援附近推薦、主動問地點、以及 `現在附近 / 等下要去的地方 / 家附近 / 公司附近` 的對話分流。

## 高頻場景

- 通勤早餐
  - 07:00-10:00
  - 常見：早餐店、超商、咖啡、飯糰、蛋餅、奶茶
- 午休快吃
  - 11:30-13:30
  - 常見：便當、自助餐、麵店、小吃、超商鮮食
- 下午嘴饞
  - 14:30-17:30
  - 常見：手搖飲、甜點、炸物小點
- 下班外帶
  - 18:00-20:30
  - 常見：便當、麵飯、火鍋、簡餐
- 夜市或宵夜
  - 20:30 之後
  - 常見：鹽酥雞、雞排、滷味、粥、超商

## 台北高頻節點

- 台北車站
- 西門
- 市政府 / 信義區
- 中山 / 南西
- 忠孝復興
- 南京復興
- 松江南京

## 推薦時的地點優先順序

1. 使用者本次明確指定的地方
2. LIFF geolocation
3. LINE location message
4. saved place
5. 系統預設地點

## 對話提示

- 問推薦時若沒有位置上下文，先問：
  - 現在附近
  - 等下要去的地方
  - 家附近
  - 公司附近
  - 我自己輸入

## 風險提醒

- 高人流商圈常伴隨甜點、飲料、速食、分享餐與加購風險。
- 夜市與攤販集中場要特別警覺少量多樣累積。
- 辦公區與通勤站點則更適合優先推 `穩定店家 + 黃金訂單`。
